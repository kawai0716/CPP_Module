# cpp7 学習ポイント（課題別）

## ex00（swap/min/max）
- 関数テンプレートの宣言と定義（`template <typename T>`）: 型に依存しない関数を1つ書いて、必要な型ごとに実体化させる考え方。
- 参照の使い方（`T&` で値を入れ替える）: 実引数そのものを入れ替えるため、参照で受けてコピーを避ける。
- const参照の使い分け（`const T&` で比較）: 比較だけなら変更しないのでconstで受けて安全にする。
- 比較演算子の前提（`<` `>` `==` が使える型）: テンプレートは演算子が使える型にしか適用できないことを理解する。
- 仕様通りの分岐（同値なら第2引数を返す）: `min/max`の細かい仕様を正確に満たす練習。

## ex01（iter）
- テンプレート関数に配列を渡す方法（`T*` と要素数）: 配列はポインタに崩れるので、先頭アドレスと長さで扱う。
- 関数ポインタの渡し方（`void (*f)(T&)` など）: 要素ごとに呼び出す関数を引数として受け取る方法を学ぶ。
- const要素への対応（`const T*` と `void (*f)(const T&)`）: const配列では要素を変更できないので、引数型を合わせる必要がある。
- オーバーロード/テンプレートの使い分け（const/非const両対応）: const/非constを両方扱える設計にする。
- ループ処理と境界（`size_t`/`unsigned int` の扱い）: 配列長を安全に回す型と境界条件を意識する。

## ex02（Array）
- クラステンプレートの宣言と実装分離（`.hpp` / `.tpp`）: テンプレートはヘッダ側に定義が必要で、分割するなら`.tpp`を使う。
- 動的配列の確保/解放（`new[]`/`delete[]`）: 要素数分を正しく確保し、必ず対応する`delete[]`で解放する。
- 例外の投げ方（範囲外で `throw std::exception`）: `operator[]`で不正アクセスを検出して例外を投げる。
- コピーコンストラクタ/代入演算子の深いコピー: コピー後に一方を変更してももう一方に影響しない設計にする。
- constメンバ関数の設計（`size() const`、`operator[] const`）: constオブジェクトからも安全に呼べるようにする。
- デフォルト初期化の挙動（`new T[n]` の意味）: 要素が型Tのデフォルト初期化になる点を理解する。
